jQuery は JavaScript のライブラリで、

ネイティブ JavaScript よりも簡単に書けることや

<p><tltp label="クロスブラウザ">どのブラウザでも同じ動作をする</tltp>であることからかなり普及してきました。</p>

<!--more-->

しかし、あまりにも普及しすぎたためか、

ネイティブの JavaScript よりも主流になった時代があったようです。

「脱 jQuery」という言葉をよく目にすることは、

jQuery は書けるのに JavaScript は書けないような人が一定数いることを示唆しています。

<br/>

## JavaScript と言いながら jQuery のコードを載せる

そういう背景があってか、jQuery のコードを載せておいて

JavaScript のコードは載せないというようなウェブサイトが多くあります。

<br/>

それだけならまだしも、**タイトルで「JavaScript で～」と言いながらコードは jQuery** だったり、

本文でも<strong>「JavaScript のコードです」と言いながら中身は jQuery のコード</strong>、なんてこともままあります。

<br/>

これはもう半分ぐらい詐欺だと思います。

それを書いている本人たちも、たぶん気づいていないんでしょうが。

<br/>

## 何が問題なのか

JavaScript と jQuery は全く別物です。

jQuery を JavaScript と呼ぶのは、東京を日本と呼ぶようなものです。

この２つの違いを知らない初心者がこのようなウェブサイトの jQuery コードを見て、

「JavaScript ってこんな書き方できるんだ～」と思ってしまうかもしれません。

そういう**無駄な誤解や混乱を生むような書き方は避けるべき**です。

<br/>

書くとしたら、「JavaScript のライブラリである jQuery を使うと」のような書き方をするか、

単に「jQuery では」とすべきです。

<br/>

## 変換ツールつくりました

こういう状況下にあるので、参考にしたいウェブサイトがあったのに jQuery で書かれていた場合、

自分で JavaScript に変換するという工程が必要になります。

jQuery を勉強したくない自分としては、手っ取り早く変換できるツールがあれば便利だなと思いました。

そこで日本語と英語で調べてみたのですが、翻訳機のような変換を提供しているサイトは見つかりませんでした。

<br/>

ということで、[こちらのサイト](https://adventar.org/calendars/1075)を参考に、変換ツールを作ってみました。

→ [jQuery -JavaScript Converter](https://jqjsconverter.netlify.app/)

これがかなり挫折と妥協の産物のようになってしまっていて、動作は中途半端なのですが、

どこかで役に立てばいいなと思います…。

これを作るために[公式ドキュメント](https://api.jquery.com/)などを読み込んだので、

だいぶ jQuery に詳しくなってしまいました。
